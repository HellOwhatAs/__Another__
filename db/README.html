第二章 关系数据库

候选码： 若关系中的某一属性组的值能唯一地标识一个元组，而其子集不能，则称该属性组为候选码。在最简单的情况下，候选码只包含一个属性。
全码： 在最极端的情况下，关系模式的所有属性组是这个关系模式的候选码，称为全码
主码： 若一个关系有多个候选码，则选定其中一个为主码
主属性： 候选码的诸属性称为主属性（Prime attribute）。
非主属性： 不包含在任何侯选码中的属性称为非主属性（Non-key attribute）。

实体完整性： 主属性不能为空
参照完整性： 外码或者为空，或者为被参照关系的主码值

第三章 关系规范化

函数依赖： “学号”函数决定“姓名”和“所在系”；“姓名”和“所在系”函数依赖于“学号”
“学号” → “姓名”；“学号” → “所在系”；
非平凡的函数依赖： 如果X→Y，但Y ⊈\nsubseteq⊈ X，则称X→Y是非平凡的函数依赖；
平凡的函数依赖： 若X→Y，但Y ⊆\subseteq⊆ X, 则称X→Y是平凡的函数依赖；
完全函数依赖： X 的任何真子集都不能函数确定 Y，记作 X→FYX\overset{F}{\to} YX→FY
部分函数依赖： X→Y 但不是完全函数依赖，记作 X→PYX\overset{P}{\to} YX→PY
传递函数依赖： X→Y(Y ⊈\nsubseteq⊈ X)，Y↛\not\to→X，Y→Z (Z ⊈\nsubseteq⊈ Y) ，则称Z对X传递函数依赖，记为X→传递\overset{\tiny\text{传递}}{\to}→传递Z。
如果 Y→\to→X,即 X↔\leftrightarrow↔Y，则X→直接\overset{\tiny\text{直接}}{\to}→直接Z

超码： 候选码是最小的超码

1NF（第一范式）： 所有属性都不可再分（不满足第一范式的数据库模式不能称为关系数据库）
2NF： 关系模式R∈1NF，并且每一个非主属性都完全函数依赖于R的码，则R∈2NF（消除部分函数依赖）
3NF： 消除非主属性对码的传递函数依赖
BCNF： 如果对于R的每个函数依赖X→Y，若Y ⊈\nsubseteq⊈ X，则X必含有（候选）码，那么R∈BCNF
（消除主属性对码的部分和传递函数依赖）
如果R∈3NF，且R只有一个候选码，则R必属于BCNF。

合并“分E-R图”时的冲突：
属性冲突： 属性域冲突，属性取值单位冲突
命名冲突： 同名异义，异名同义

第六章 数据库安全

自主存取控制（DAC）：

GRANT <权限>[,<权限>]...
[ON <对象类型> <对象名>]
TO <用户>[,<用户>]...
[WITH GRANT OPTION];

GRANT ALL PRIVILIGES /*所有权限*/
ON TABLE Student, Course
TO PUBLIC /*全体用户*/
WITH GRANT OPTION; /*将对指定操作对象的指定操作权限授予指定的用户*/

(不允许循环授权)

GRANT UPDATE(Sno), SELECT
ON TABLE Student
TO U4;

收回权限：

REVOKE <权限>[,<权限>]...
ON <对象类型> <对象名>
FROM <用户>[,<用户>]...;

REVOKE INSERT
ON TABLE SC
FROM U5 CASCADE;

CREATE USER <username>
[WITH] [DBA | RESOURCE | CONNECT];

数据库角色：

CREATE ROLE R1;

GRANT SELECT, UPDATE, INSERT
ON TABLE Student TO R1;

GRANT R1 TO 王平, 张明, 赵玲;

REVOKE R1 FROM 王平;

角色的权限修改：

REVOKE SELECT ON TABLE Student
FROM R1

强制存取控制（MAC）：
仅当主体的许可证级别大于或等于客体的密级时，该主体才能读取相应的客体
仅当主体的许可证级别小于等于客体的密级时，该主体才能写相应的客体

审计： 启用一个专用的审计日志（Audit Log）将用户对数据库的所有操作记录在上面

AUDIT ALTER, UPDATE ON SC; /*设置审计功能*/

NOAUDIT ALTER, UPDATE ON SC; /*取消审计*/

每个职工只对自己的记录有SELECT权限

GRANT SELECT ON 职工 WHEN USER() = NAME TO ALL;
